kakao_together_clone
==========================
> 카카오 같이가치의 다양한 기능을  클론 코딩한 프로젝트
#### Environment
- Windows 10 64bit
- Spring Framework 5.2.1, MyBatis
- Apache Tomcat 9.0
- JSTL, servlet-api, spring-security-web, jackson, commons-fileupload, XSS

#### purpose
> 네이버·카카오·라인·쿠팡·배달의민족 5대 it 기업 중 카카오 웹페이지를 구현하기를 통해
상용화된 기술 습득 및 응용하기 위함이며, 이는 카카오의 카카오 같이가치 기부하기를
레퍼런스 하는 것이 가장 적합하다고 판단하였습니다.    

>레퍼런스를 통해서 카카오의 장점을 파악할 수 있었고 이는 사용자 중심의 웹페이지
구현에 많은 도움이 될 거라는 판단을 하였습니다.
또한 웹페이지를 통해서 기획자의 구성을 배울 수 있었으며, 이를 구현시키기 위해서
다양하고 창의적인 사고방식을 훈련하게 하였습니다.   

>또한 it 선도기업의 웹페이지는 웹트렌드를 읽기에는 가장 최적화된 사이트라고 판단하여
변화하는 웹 트렌드를 읽고 배우고 싶었습니다.

전체 공통기능

- xss 사용 규격외 입력 방지

이승대
--------------------------
### User
#### 회원가입 
- 가입란 미입력시 등록 방지
- 입력란 유효성 검사(Alert 및 focus로 이동방지)  
- Ajax 사용 입력란 유효성 중복 체크  
- 비밀번호 이중 체크
- 아이디 중복 방지
- 다음 지도 API 사용 주소 입력
- 관리자 계정(아이디 명 Admin은 가입 불가)
- BCrypt 비밀번호 암호화

#### 로그인  
- 로그인 체크 및 세션 유지 구현  
- 로그아웃시 세션 종료
- 아이디 및 비밀번호 일치 확인 추가
#### 마이페이지
- 다음 지도 API 이용 주소 등록 가능
- 회원가입에 사용된 기능과 동일
- 저장하기 클릭 시 수정된 내용 저장
- 탈퇴하기 클릭 시 DB에서 정보 삭제 

####  KaKao API & Naver API 
- 네이버 로그인 API 구현  
- 카카오 로그인 API 및 카카오 로그인 데이터베이스 구현  
- 카카오계정 및 세션 동시 로그아웃 구현

김준오
---------------------------
#### 공통 기능
- 게시글이 없을 경우 표현
- 베너 및 슬라이드 클릭시 해당하는 페이지로 이동
- header 및 footer 구현
- 각 페이지 타이틀 변경(Favicon 사용)
#### 메인 페이지
- Slick slider 사용, 오토슬라이드, 버튼식 슬라이드 구현
- 점자 버튼 및 삼각 버튼 클릭시 이전, 다음 내용 표현
- 최대 게시글 표현 제한(최신순 8개)
- 더보기 클릭 시 같이기부 페이지로 이동
- 회원 로그인 시 제안하기 / 관리자 로그인 시 제안받기로 표현
- 같이 기부 총 금액 및 현재날짜 표현

#### 제안
- 입력란 유효성 검사(Alert 및 focus로 이동방지)  
- Ckeditor 사용, 이미지 업로드 및 문단 구분 기능 구현 
- 제안하기 클릭 시 관리자 페이지(제안받기)로 송출
- 제안받기 클릭 시 제안하기 에서 송출된 내용 자동으로 입력
- 기부제안 및 행동제안 구현, 등록 시 해당 페이지에 표현


#### 같이기부 페이지
- 제안하기에 입력된 종료날짜 기준, 초과시 자동으로 모금 후기 이동
- Ajax사용 , 태그별 페이지 이동 처리,  태그안의 태그에서도 같은 기능 수행  
(ex. 모금중-전체,어린이,동물....-추천순,최신순 // 모금후기-전체,어린이.....-추천순,최신순)
- 목표금액 및리스트 하단에 출력
- 동적쿼리 사용 각 페이지 리스트 출력
#### 행동하기 페이지
- 해당 영역 클릭 시 페이지 이동
- 이외 내용 같이기부와 동일


김세호
--------------------------
#### 공통사항
- textarea 입력한 글자 수 표현(KeyPress 사용)
- textarea 입력 초과 방지 및 미입력시 등록 방지
- 금액 처리는 정규식 사용 3자리씩 , 로 전환
- 이벤트 참여 당사자에게만 하트 활성화/비활성화 표현
- 참여 당사자 이외에는 카운트 증가만 표현  
 (ex. A유저 참여시 하트는 활성화 상태,B 유저 관점에서는 비활성화 상태로 표현)
- 해당 게시글, ID, 댓글 삭제시 연계된 기능 DB 정보 삭제
- Ajax 사용 전체페이지 새로고침 없이 기능 수행

### TotalDonation
#### 기부 금액
- 전체적인 금액 추가 / 감소
#### 애니메이션
- 기부 금액 %(총 금액/목표금액)에 따라 애니메이션 이동
### Comment
#### 기부금액
- 직접기부 및 댓글기부 금액을 합산 표현
- 댓글기부는 1회만 지급(지급 후에는 댓글 기부 참여 완료 로 표현)
#### 댓글
- 댓글 카운트 기능 
- 댓글이 없을 경우 표현
- 아이디 3글자 이외 마스킹 처리
- 등록시간 표현(ex.방금전, 9분전,1시간전 // 1달 이후에는 2022.07.16 등과 같이 표현)
- 본인 작성 댓글은 삭제, 타인 작성 댓글은 신고
#### 더보기
- 댓글이 5개 이상일 시 더보기 버튼 표현
- 더보기 클릭 시 5개씩 출력
- 댓글 최대 출력 후 더보기 버튼 숨김(.hide)
### UnderBar
#### UnderBar
- 언더바는 휠의 위치에 따라 이동
- footer 영역 도달 시 하단에 고정
#### 응원하기 & 공유하기
- 응원하기 및 공유하기 버튼 클릭 시 기부금 100원 추가(페이지당 1회 한정)
- 응원하기 클릭 시 하트 색 변경(취소 불가능)
- 공유하기 클릭 시 URL 복사
#### 기부하기
- 기부하기 클릭 시 팝업창 생성, 팝업창 이외 배경은 불투명하게 전환
- 지정된 금액 클릭으로 조합 가능, 다시입력 클릭 시 초기화
- 조합된 금액 표현 및 결제한 금액 댓글창에 표현
- 결제하기 클릭 시 댓글과 동일한 기능으로 등록
### Heart
#### Heart
- 좋아요 카운트 기능
- 클릭 으로 활성화, 비활성화 가능
